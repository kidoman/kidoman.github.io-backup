.NET Goes Native

May 10th, 2014

.NET (specifically C#) was my first love. I discovered it 2002 shortly after its release, and it felt like a breath of fresh air. This is even before C# got generics. The tooling and the awesome IDE (i.e. Visual Studio) were big contributors to the attraction of the platform.

.NET has improved a lot since then. C# has always felt like a language which was in tune with the times (unlike some other leading platforms.) Built in support for the async / await keywords is one example of how the C# language has been evolved to keep it relevant with the changing landspace of system programming.

Going Native

The recent additions to the .NET framework / toolchain shows a hunger for change / improvement from a newly reinvigorated Microsoft. The Roslyn compilers are a big step forward.

The .NET native toolchain appears to be the other big improvement. A C# program which is converted to a single static executable seems like a dream come true. No need to install the right version of .NET on the target, just ensure that the right version of the binary is copied over and voila.

I hope that someday the flagship Visual Studio IDE will be available for development on all major operating systems (Macs / Linux.)
